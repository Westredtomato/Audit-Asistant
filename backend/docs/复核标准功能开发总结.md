# 复核标准功能开发总结

## 项目概述

为"注会帮"审计复核辅助软件开发了完整的**复核标准管理系统**，实现了复核标准的智能存储、语义检索和AI生成功能，支持审计工作的标准化和智能化。

## 完成的功能模块

### 1. 数据库设计 ✅

**文件**: `backend/app/models/review_standard.py`

- **ReviewStandard**: 复核标准主表

  - 存储标题、审计结论、证据分类等信息
  - 支持分类、标签、来源等元数据
  - 记录使用次数和时间戳
  - 与用户、项目建立关联关系
- **ReviewStandardVector**: 向量索引表

  - 存储审计结论的文本向量
  - 支持 1024 维向量存储（JSON格式）
  - 记录嵌入模型信息

### 2. 数据模型定义 ✅

**文件**: `backend/app/schemas/review_standard.py`

定义了完整的请求/响应数据模型：

- `ReviewStandardCreate/Update/Response`: 标准的CRUD模型
- `SemanticSearchRequest/Response`: 语义检索模型
- `GenerateStandardRequest/Response`: AI生成模型
- `ReviewStandardStats`: 统计信息模型

### 3. 信息检索服务 ✅

**文件**: `backend/business/retrieval_service.py`

#### 核心类

- **EmbeddingService**: 文本向量化服务

  - 调用阿里云通义千问 `text-embedding-v3` 模型
  - 支持单个和批量文本向量化
  - 生成 1024 维向量
- **VectorSearchService**: 向量检索服务

  - 实现余弦相似度计算
  - 实现欧氏距离计算
  - 支持相似度阈值过滤
- **RetrievalService**: 信息检索服务

  - 为复核标准创建向量索引
  - 执行语义相似度检索
  - 批量更新向量索引

#### 核心功能

```python
# 语义检索
result = await retrieval_service.semantic_search(
    SemanticSearchRequest(
        query="审计结论文本",
        top_k=5,
        similarity_threshold=0.7
    )
)
```

### 4. AI生成服务 ✅

**文件**: `backend/business/standard_generation_service.py`

#### StandardGenerationService

- 基于审计结论和参考标准生成复核标准
- 使用 LLM（qwen3-plus）生成结构化数据
- 支持自动检索参考标准
- 支持根据反馈重新生成

#### 核心功能

```python
# AI 生成复核标准
result = await generation_service.generate_standard(
    GenerateStandardRequest(
        audit_conclusion="审计结论",
        reference_standards=[1, 2, 3],
        auto_save=True
    )
)
```

#### 技术特点

- 智能提示词工程，引导 LLM 生成规范的 JSON 格式
- 自动提取 JSON 内容，处理多种格式
- 参考已有标准，提高生成质量
- 支持自动保存和向量索引创建

## 技术架构

### 数据流

```
用户输入审计结论
    ↓
调用 EmbeddingService
    ↓
生成 1024 维向量
    ↓
VectorSearchService 计算相似度
    ↓
返回 Top-K 相似结果
    ↓
（可选）作为参考标准
    ↓
调用 StandardGenerationService
    ↓
LLM 生成复核标准
    ↓
保存到数据库 + 创建向量索引
```

### 关键技术

1. **文本向量化**

   - 模型: `text-embedding-v3`
   - 维度: 1024
   - 提供商: 阿里云通义千问
2. **相似度计算**

   - 算法: 余弦相似度（Cosine Similarity）
   - 范围: 0-1（值越大越相似）
   - 默认阈值: 0.7
3. **AI 生成**

   - 模型: `qwen3-plus`
   - 温度: 0.7
   - 最大token: 4000
   - 输出格式: 结构化 JSON
4. **数据存储**

   - 数据库: SQLite
   - 向量存储: JSON 格式
   - ORM: SQLAlchemy

## 使用前提

### 必需配置

1. **API Key**: 必须配置阿里云 API Key

   ```python

   # 或通过环境变量
   export API_KEY="your-api-key"
   ```
2. **依赖包**: 已在 `requirements.txt` 中

   - `openai`: LLM 调用
   - `httpx`: HTTP 客户端
   - `numpy`: 向量计算

- `BASE_URL`: LLM API 地址（默认阿里云）

- 相似度阈值、Top-K 等参数

- **当前方案**: 全量扫描 + 相似度计算
- **适用规模**: < 1000 条标准

## 注意事项

1. **API Key 安全**: 不要将 API Key 提交到代码仓库
2. **向量索引**: 修改审计结论后需要更新向量索引
3. **权限控制**: 只有创建者和管理员可以修改标准
4. **成本控制**: LLM 调用和向量化有 API 成本
5. **并发处理**: 向量化和 LLM 调用需要处理并发限制

## 问题排查

### 问题 1: 语义检索无结果

**可能原因**:

- 查询文本过短
- 相似度阈值过高
- 缺少向量索引

**解决方案**:

```bash
# 重建向量索引
curl -X POST http://localhost:8000/api/v1/review-standards/batch/rebuild-embeddings \
  -H "Authorization: Bearer $TOKEN"
```

### 问题 2: AI 生成失败

**可能原因**:

- API Key 未配置或无效
- 网络连接问题
- LLM 服务不可用

**解决方案**:

- 检查 API Key 配置
- 检查网络连接
- 查看日志输出

### 问题 3: 向量索引创建失败

**可能原因**:

- API Key 问题
- 文本过长或包含特殊字符

**解决方案**:

- 检查审计结论文本
- 查看详细错误日志
- 手动重建索引
