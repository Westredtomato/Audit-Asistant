# 复核标准功能使用说明

## 功能概述

复核标准管理系统提供了以下核心功能：

1. **复核标准存储**：将审计复核标准存储在数据库中，支持结构化的证据分类与质量要求
2. **语义检索**：基于审计结论进行语义相似度检索，快速找到相关的复核标准
3. **智能生成**：使用 AI 根据审计结论自动生成复核标准
4. **标准管理**：支持创建、更新、删除、查询复核标准

## 数据结构


## API 端点说明

### 1. 创建复核标准

**端点**: `POST /api/v1/review-standards/`

**请求体**:

```json
{
  "title": "标准标题",
  "audit_conclusion": "审计结论文本",
  "evidence_classification": [...],
  "category": "分类标签（可选）",
  "tags": "标签1,标签2（可选）",
  "source": "user_created",
  "project_id": 1
}
```

**响应**: 返回创建的复核标准对象

**说明**: 创建后会自动生成向量索引用于语义检索

---

### 2. 语义检索复核标准

**端点**: `POST /api/v1/review-standards/search/semantic`

**请求体**:

```json
{
  "query": "审计结论文本（至少10个字符）",
  "top_k": 5,
  "similarity_threshold": 0.7,
  "category": "可选的分类筛选",
  "project_id": null
}
```

**响应**:

```json
{
  "query": "查询文本",
  "results": [
    {
      "standard_id": 1,
      "title": "标准标题",
      "audit_conclusion": "审计结论",
      "evidence_classification": [...],
      "similarity_score": 0.95,
      "category": "分类",
      "tags": "标签",
      "usage_count": 10
    }
  ],
  "total_found": 5
}
```

**说明**:

- `similarity_score`: 0-1 之间，值越大表示越相似
- 默认返回相似度 ≥ 0.7 的结果
- 按相似度从高到低排序

---

### 3. AI 生成复核标准

**端点**: `POST /api/v1/review-standards/generate`

**请求体**:

```json
{
  "audit_conclusion": "审计结论文本（至少20个字符）",
  "reference_standards": [1, 2, 3],
  "auto_save": true,
  "project_id": 1,
  "title": "自定义标题（可选）"
}
```

**响应**:

```json
{
  "审计结论": "审计结论",
  "审计证据分类与要求": [...],
  "generated_title": "自动生成的标题",
  "reference_count": 3,
  "saved": true,
  "saved_id": 10
}
```

**说明**:

- `reference_standards`: 可选，提供参考标准的 ID 列表
- 如果不提供参考标准，系统会自动进行语义检索找到相似标准作为参考
- `auto_save`: 是否自动保存生成的标准到数据库

---

### 4. 获取复核标准列表

**端点**: `GET /api/v1/review-standards/`

**查询参数**:

- `page`: 页码（默认 1）
- `page_size`: 每页数量（默认 20，最大 100）
- `category`: 分类筛选
- `source`: 来源筛选（user_created/llm_generated/system_preset）
- `project_id`: 项目筛选
- `search`: 关键词搜索

**响应**:

```json
{
  "total": 100,
  "items": [...],
  "page": 1,
  "page_size": 20
}
```

---

### 5. 获取单个复核标准

**端点**: `GET /api/v1/review-standards/{standard_id}`

**响应**: 返回指定的复核标准对象

**说明**: 每次访问会增加该标准的使用计数

---

### 6. 更新复核标准

**端点**: `PUT /api/v1/review-standards/{standard_id}`

**请求体**: （所有字段可选）

```json
{
  "title": "新标题",
  "audit_conclusion": "新的审计结论",
  "evidence_classification": [...],
  "category": "新分类",
  "tags": "新标签"
}
```

**说明**:

- 只有创建者或管理员可以修改
- 如果修改了 `audit_conclusion`，会自动更新向量索引

---

### 7. 删除复核标准

**端点**: `DELETE /api/v1/review-standards/{standard_id}`

**响应**: 204 No Content

**说明**:

- 只有创建者或管理员可以删除
- 删除时会同时删除关联的向量索引

---

### 8. 获取统计信息

**端点**: `GET /api/v1/review-standards/stats/overview`

**响应**:

```json
{
  "total_standards": 100,
  "by_category": {
    "银行存款": 20,
    "收入确认": 30
  },
  "by_source": {
    "user_created": 50,
    "llm_generated": 30,
    "system_preset": 20
  },
  "most_used": [...],
  "recent_created": [...]
}
```

---

### 9. 重建向量索引（管理员）

**端点**: `POST /api/v1/review-standards/batch/rebuild-embeddings`

**响应**:

```json
{
  "message": "向量索引重建完成",
  "stats": {
    "total": 100,
    "success": 95,
    "failed": 0,
    "skipped": 5
  }
}
```

**说明**:

- 仅管理员可执行
- 用于初始化或修复向量索引

## 使用流程示例

### 场景 1: 用户输入审计结论，生成复核标准

```python
# 1. 用户输入审计结论
audit_conclusion = "经审计，被审计单位的收入确认时点..."

# 2. 调用语义检索 API，查找相似标准
search_response = requests.post(
    "http://localhost:8000/api/v1/review-standards/search/semantic",
    json={
        "query": audit_conclusion,
        "top_k": 3,
        "similarity_threshold": 0.7
    },
    headers={"Authorization": f"Bearer {token}"}
)

# 3. 提取参考标准 ID
reference_ids = [r["standard_id"] for r in search_response.json()["results"]]

# 4. 调用生成 API，基于参考标准生成新标准
generate_response = requests.post(
    "http://localhost:8000/api/v1/review-standards/generate",
    json={
        "audit_conclusion": audit_conclusion,
        "reference_standards": reference_ids,
        "auto_save": True,
        "project_id": 1
    },
    headers={"Authorization": f"Bearer {token}"}
)

# 5. 获取生成的标准
generated_standard = generate_response.json()
print(f"生成的标准ID: {generated_standard['saved_id']}")
```

### 场景 2: 直接生成（不提供参考标准）

```python
# 系统会自动检索相似标准作为参考
generate_response = requests.post(
    "http://localhost:8000/api/v1/review-standards/generate",
    json={
        "audit_conclusion": "经审计，被审计单位的关联方交易...",
        "auto_save": False  # 不自动保存，让用户预览
    },
    headers={"Authorization": f"Bearer {token}"}
)

generated = generate_response.json()
# 用户确认后再手动保存
```

### 场景 3: 查询和使用已有标准

```python
# 1. 按分类查询标准
list_response = requests.get(
    "http://localhost:8000/api/v1/review-standards/",
    params={
        "category": "银行存款",
        "page": 1,
        "page_size": 10
    },
    headers={"Authorization": f"Bearer {token}"}
)

# 2. 获取具体标准详情
standard_id = list_response.json()["items"][0]["id"]
detail_response = requests.get(
    f"http://localhost:8000/api/v1/review-standards/{standard_id}",
    headers={"Authorization": f"Bearer {token}"}
)

standard = detail_response.json()
# 使用标准进行复核
```

## 技术实现说明

### 1. 向量嵌入

- 使用阿里云通义千问的 `text-embedding-v3` 模型
- 将审计结论文本转换为 1024 维向量
- 向量存储在 `review_standard_vectors` 表中

### 2. 相似度计算

- 使用余弦相似度（Cosine Similarity）
- 相似度范围: 0-1，值越大表示越相似
- 默认阈值: 0.7

### 3. AI 生成

- 使用阿里云通义千问 `qwen3-plus` 模型
- 基于参考标准和提示词生成结构化的复核标准
- 支持迭代优化和反馈调整

### 4. 数据持久化

- 使用 SQLite 数据库存储复核标准
- 向量数据以 JSON 格式存储
- 支持增量更新和批量重建索引

## 配置说明

在 `backend/business/system_config_service.py` 中配置：

```python
class Settings(BaseSettings):
    # LLM 配置
    API_KEY: Optional[str] = None  # 必须配置
    BASE_URL: str = "https://dashscope.aliyuncs.com/compatible-mode/v1"
    LLM_MODAL: str = "qwen3-plus"
```

**重要**: 使用前必须配置 `API_KEY`，可以通过环境变量或 `.env` 文件设置。

## 注意事项

1. **API Key**: 必须配置有效的阿里云 API Key
2. **向量索引**: 新创建的标准会自动生成向量索引，修改审计结论后需要更新索引
3. **性能**: 大量标准的语义检索可能需要较长时间，建议适当设置 `top_k` 值
4. **权限**: 只有创建者和管理员可以修改/删除标准
5. **初始化**: 系统启动时会自动加载 3 个预置的示例标准
