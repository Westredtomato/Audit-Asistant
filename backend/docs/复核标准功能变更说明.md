### 3. 公共API接口变更 ⚠️

**新增路由组**: `/api/v1/review-standards`

| 方法   | 路径                          | 功能                             |
| ------ | ----------------------------- | -------------------------------- |
| POST   | `/search/semantic`          | **语义检索**（核心接口）   |
| POST   | `/generate`                 | **AI生成标准**（核心接口） |
| POST   | `/`                         | 创建复核标准                     |
| GET    | `/`                         | 获取标准列表                     |
| GET    | `/{id}`                     | 获取单个标准                     |
| PUT    | `/{id}`                     | 更新标准                         |
| DELETE | `/{id}`                     | 删除标准                         |
| GET    | `/stats/overview`           | 统计信息                         |
| POST   | `/batch/rebuild-embeddings` | 重建索引（管理员）               |

### 4. 数据库变更

- **新增表**: `review_standards`, `review_standard_vectors`
- **修改表**: `users`, `projects`（新增关联关系）

### 5. 系统配置变更

- **必须配置**: `API_KEY`（阿里云通义千问API密钥）
- 配置位置: `backend/business/system_config_service.py`

## 二、协调

### 1. 前端

**需要协调的核心接口调用**:

```javascript
// 步骤1: 语义检索（可选）
POST /api/v1/review-standards/search/semantic
{
  "query": "用户输入的审计结论",
  "top_k": 3,
  "similarity_threshold": 0.7
}

// 步骤2: AI生成复核标准（耗时10-30秒，需显示进度）
POST /api/v1/review-standards/generate
{
  "audit_conclusion": "用户输入的审计结论",
  "reference_standards": [1, 2, 3],  // 来自步骤1
  "auto_save": true,
  "project_id": 当前项目ID
}

// 响应数据结构
{
  "审计结论": "...",
  "审计证据分类与要求": [
    {
      "一级分类": {
        "二级分类": [
          {
            "证据内容": "...",
            "质量要求": "..."
          }
        ]
      }
    }
  ],
  "saved_id": 10
}
```

- 生成进度显示（10-30秒加载动画）
- 显示"复核标准"文件入口
- 右侧边栏展示复核标准内容

## 四、核心API接口详细说明

### 接口1: 语义检索

```
POST /api/v1/review-standards/search/semantic
```

**请求**:

```json
{
  "query": "审计结论文本",
  "top_k": 5,
  "similarity_threshold": 0.7
}
```

**响应**:

```json
{
  "results": [
    {
      "standard_id": 1,
      "title": "标题",
      "similarity_score": 0.95,
      "evidence_classification": [...]
    }
  ]
}
```

---

### 接口2: AI生成复核标准

```
POST /api/v1/review-standards/generate
```

**请求**:

```json
{
  "audit_conclusion": "审计结论文本",
  "reference_standards": [1, 2, 3],
  "auto_save": true,
  "project_id": 1
}
```

**响应**:

```json
{
  "审计结论": "...",
  "审计证据分类与要求": [...],
  "saved": true,
  "saved_id": 10
}
```

**注意**: 耗时10-30秒，前端需显示加载进度

## 五、配置说明

### API_KEY配置（必需）

```bash
# Windows PowerShell
$env:API_KEY="your-api-key"

# Linux/Mac
export API_KEY="your-api-key"

# 或在 backend/business/system_config_service.py 中配置
API_KEY = "your-api-key"
```

**未配置的影响**:

- ❌ 无法使用语义检索
- ❌ 无法使用AI生成
- ✅ 可以手动管理标准
